Installation Steps

Laraship Structure

  • Root Folder
  • App Folder which contains models, controllers, middleware, and Requests
  • Resources Folder which contains views and Layouts
  • Public Folder which contains assets, plugins and uploads

Requirements

Laraship is built on Laravel with Stripe so the following Requirements are needed

  • Composer
  • PHP >= 5.5.9
  • OpenSSL PHP Extension
  • PDO PHP Extension
  • Mbstring PHP Extension
  • Tokenizer PHP Extension
  • Stripe Account

Composer

  • Laravel utilizes Composer to manage its dependencies. So, before using Laravel, make sure you have Composer installed on your machine.
  • In Laraship root folder run the command 
    project_root> Composer install
    chmod -R 0777 storage/*

Database and Configuration

  • You can find .env example file in the Laraship package root
  • You must have a database with connection and authentication details
  • In .env file or in Laravel configuration file directly set 
    DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laraship
DB_USERNAME=username
DB_PASSWORD=password
  • In Laraship root folder run the following commands 
    project_root> php artisan key:generate
                    
    project_root> php artisan migrate

Stripe Hooks

  • Webhooks are configured in the webhooks settings section of the Dashboard. Clicking Add endpoint reveals a form to add a new URL for receiving webhooks.

    Stripe supports two webhook types: Account and Connect. You'll likely want to create an account webhook, unless you've created a Connect application.

    You can enter any URL you'd like to have receive the events, but this should be a dedicated page on your server, coded per the instructions below. The mode determines whether test events or live events are sent to this URL; if you want to send both live and test events to the same URL, you need to create two separate settings. You may add as many URLs as you like.

    You can also choose to be notified of all event types, or only specific ones.
